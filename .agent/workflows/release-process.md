# PGVectorRAGIndexer Release Process

1. **Releasing the App**
   - ALWAYS use the `./release.sh` script to bump versions and create the tag.
   - Example to bump patch version: `./release.sh patch`
   - Example without prompting: `./release.sh -y patch`

2. **Wait for MSI Pipeline**
   - The `./release.sh` push to GitHub will automatically trigger the `Build Windows Installer` GitHub Actions workflow.
   - Wait for it to finish and produce the unsigned MSI artifact.

3. **Download Unsigned MSI**
   - Download the new MSI (`PGVectorRAGIndexer.msi`) generated by the pipeline to the exact folder:
     `C:\Users\v_ale\Desktop\ToSign\PGVectorRAGIndexer-unsigned`

4. **Code Signing**
   - Prompt the user to run their `signtool` command.
   - Wait patiently for the user to explicitly tell you that the MSI has been signed successfully.

5. **Upload to GitHub Release**
   - Take the signed MSI from `C:\Users\v_ale\Desktop\ToSign\PGVectorRAGIndexer-unsigned\PGVectorRAGIndexer.msi`.
   - Upload it to the new GitHub Release using the GitHub CLI (`gh release upload`).
   - Example: `gh release upload v{VERSION} /mnt/c/Users/v_ale/Desktop/ToSign/PGVectorRAGIndexer-unsigned/PGVectorRAGIndexer.msi --clobber`
